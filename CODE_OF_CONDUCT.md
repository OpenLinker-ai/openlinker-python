# Code of Conduct

OpenLinker projects should remain useful, respectful, and focused on the work.

## Expected Behavior

- Be direct, specific, and constructive.
- Keep criticism about code, documentation, or decisions rather than people.
- Respect differences in experience, language, and background.
- Protect private information, credentials, user data, and vulnerability details.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing another person's private information.
- Posting live secrets, user data, or uncoordinated exploit details.
- Repeatedly derailing technical discussions after maintainers have redirected
  them.

Report conduct concerns through the project's published support channel. Report
vulnerabilities through [SECURITY.md](./SECURITY.md), not a public issue.

Maintainers may edit or remove content and limit participation when necessary to
keep the project safe and workable.
